# The single `release.yml` model

`otf-release` replaces the changesets-era setup (a separate `workflow_dispatch` plus a
`private:true` guard) with **one** workflow and a proper dependency DAG. The workflow is
generated by [`init`](./commands/init.md); this doc explains its shape and the reasoning.

## The shape

```
build-matrix  (asset packages only)  ──needs──▶  publish  (ALL packages, topological order)
```

- **`build-matrix`** — present only if any [asset packages](./README.md#glossary) exist.
  Cross-compiles the configured target triples and uploads the binaries as artifacts.
- **`publish`** — `needs: build-matrix`; downloads artifacts into `.artifacts/`; runs
  `otf-release publish`, which releases the **entire topological set** (libraries *and* asset
  packages) in dependency order, attaching staged binaries where they exist on disk.

Trigger: a merge to `main` (i.e. merging a release PR produced by
[`version`](./commands/version.md)).

## Why one workflow

| Property | How the single model gives it |
| --- | --- |
| **All-or-nothing** | If any matrix leg fails, `publish` never runs. Intrinsic to `needs:`, not bolted on. |
| **Correct ordering** | "Asset package depends on freshly-published libs" is solved by topological publish in **one run** — libs first, asset package after. No manual two-phase, no "run after libs released." |
| **No guard hack** | Asset packages are first-class publishable packages with a binary target. No `private:true`, no flip-off step. |
| **Stateless** | `publish` reads what to attach from `.artifacts/<pkg>/` on disk — nothing persisted. |

## npm specifics

Secrets/auth: `NODE_AUTH_TOKEN`. (cargo would add `CARGO_REGISTRY_TOKEN` later.)

**Gotchas to keep** (see [adapters/npm.md](./adapters/npm.md)):

- Idempotent `npm view` skip → resumable after a partial failure.
- `--no-workspaces` → the private root workspace would otherwise skip packages.
- `--access public` → scoped package first publish.
- Brotli compression via Node `zlib` → no runner-side CLI dependency.

**Gotcha to drop:** the `private:true` guard flip, entirely.

## The generated file is yours

`init` emits an **editable scaffold**. Matrix triples carry a `# edit me` marker, and
repo-specific build steps are expected to be added by hand. Re-running `init` warns before
overwrite (`--force` to replace) and does not try to manage the file after generation.
